How to copy LOCAL Docker Postgres data → Railway Postgres
==========================================================

Important
---------
• git push / Railway redeploy only updates CODE. It does NOT copy your laptop database rows.
• Remote tables must exist first: backend must deploy successfully so "prisma migrate deploy" runs on Railway.

Step A — Railway has tables
----------------------------
Redeploy the API service (GitHub push). In logs you should see migrations then "listening".

Step B — Export from your Mac (local Docker DB running)
--------------------------------------------------------
From repo root (folder that contains docker-compose.yml):

  docker compose up -d postgres
  ./AirShipBackend/scripts/export-local-db.sh

Creates: airship-data-only.sql (in repo root)

Step C — Railway PUBLIC database URL (not .railway.internal)
--------------------------------------------------------------
1. Railway dashboard → Postgres service (database plugin).
2. Find PUBLIC / external / TCP proxy connection string.
   It must NOT contain "postgres.railway.internal" (that only works inside Railway).
3. Copy the full postgresql://... URL.

Step D — Import from your Mac
------------------------------
Install psql if needed:  brew install libpq   then follow brew's hint to link psql.

  cd /path/to/AirShip    (repo root)
  export DATABASE_URL='paste-public-url-here'

Run import (script adds sslmode=require if missing):

  ./AirShipBackend/scripts/import-to-hosted-db.sh ./airship-data-only.sql

Step E — Check API
------------------
Open in browser or curl:

  https://airship-app-production.up.railway.app/api/cities

You should see your imported rows.

If import fails
----------------
• "relation does not exist" → migrations never ran on Railway; fix deploy first.
• "duplicate key" → hosted DB already had rows; clear hosted data only if you intend to replace (advanced).
• Connection timeout → wrong URL, firewall, or need public networking enabled on Railway Postgres.
